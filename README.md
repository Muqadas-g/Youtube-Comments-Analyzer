---

## 📌 Project Overview

This project performs sentiment analysis on real-world YouTube comments collected and labeled manually. Unlike standard benchmark datasets, this project uses self-collected, domain-specific data — making it a true end-to-end data science workflow.

**DecodeLabs Data Science Internship | Project 3**

---

## 🏗️ Pipeline Architecture
YouTube Comments → Manual Labeling → Pre-Processing → Vectorization → ML Model → LLM Extension
---

## ✨ What Makes This Different

- **Real self-collected data** — Comments manually gathered and labeled, not a pre-packaged dataset
- **Dual approach** — Classical ML (Scikit-learn) + Local LLM (Llama 3.2 via Ollama)
- **Public dataset published** — Dataset available on Kaggle for community use
- **End-to-end ownership** — Data collection, cleaning, modeling, and deployment all done independently

---

## ⚙️ Tech Stack

| Library | Purpose |
|---------|---------|
| NLTK / SpaCy | Text pre-processing |
| Scikit-learn | Classical ML classification |
| Llama 3.2 (Ollama) | Local LLM sentiment inference |
| Pandas | Data manipulation |
| Matplotlib | Visualization |
| Kaggle | Notebook + Dataset hosting |

---

## 📊 Key Steps

1. **Data Collection** — YouTube comments manually collected and labeled
2. **Pre-Processing** — Tokenization, stop-word removal, lemmatization
3. **Vectorization** — TF-IDF feature extraction
4. **Classification** — Sentiment prediction (Positive / Negative)
5. **LLM Extension** — Llama 3.2 via Ollama for deep language understanding
